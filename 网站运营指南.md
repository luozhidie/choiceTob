## 一、网站概述

### 1.1 网站定位

骆芷蝶智选是一个面向服装零售店主的智能化选品与企划服务平台,提供:

- **买手选品**: 批发价商品采购
- **商品企划**: AI智能生成商品企划报告
- **陈列搭配**: 科学的陈列方案和每日搭配灵感
- **风格测试**: 在线色彩季型和风格测试工具
- **会员服务**: 多层级会员体系和付费订阅

### 1.2 核心技术栈

- **前端**: Next.js 16 (App Router) + React 19 + TypeScript
- **后端**: Next.js API Routes + Supabase (PostgreSQL)
- **支付**: 微信支付(JSAPI + Native扫码)
- **部署**: Vercel (自动部署from GitHub main分支)
- **数据库**: Supabase (包含认证、数据库、存储)

## 三、支付流程详解

### 3.1 支付架构

**微信支付集成方式**:
- **JSAPI支付**: 在微信内直接拉起支付(需要openid)
- **Native支付**: 生成支付二维码,用户扫码支付
- **支付平台判断**:
  ```javascript
  const isWeChat = /MicroMessenger/i.test(navigator.userAgent);
  if (isWeChat) {
    // 在微信内,直接跳转微信支付链接
    window.location.href = result.code_url;
  } else {
    // 非微信内,显示支付二维码
    setPayQrCode(result.code_url);
    setShowPayModal(true);
  }
  ```

### 3.2 各功能支付流程

#### 3.2.1 商品购买 (checkout/page.tsx)
1. 用户点击"立即下单"
2. 填写收货信息
3. 点击"立即支付"
4. 调用 `/api/wechat-pay/unified-order` API
5. 根据环境跳转到微信支付或显示二维码
6. 支付成功后,轮询订单状态
7. 订单状态变为`confirmed`后,跳转成功页

#### 3.2.2 VIP会员订阅 (vip/page.tsx)
1. 用户选择会员套餐
2. 点击"立即订阅"
3. 创建`membership_orders`记录(status=`pending`)
4. 调用微信支付API
5. 支付成功后,管理员在后台确认
6. 确认后,用户会员权限生效

#### 3.2.3 每日搭配灵感订阅 (display/page.tsx)
1. 用户点击"立即解锁"
2. 选择订阅周期(月度/年度)
3. 创建`membership_orders`记录(plan_id=`daily_looks`)
4. 调用微信支付API
5. 支付成功后,更新订阅状态
6. 用户可查看完整搭配方案

#### 3.2.4 商品企划支付 (planning-tool/page.tsx)
1. 用户点击"立即支付 ¥2980"
2. 创建`planning_payments`记录(status=`pending`)
3. 调用微信支付API
4. 支付成功后,status更新为`paid`
5. 用户进入填资料页面
6. 提交资料后,AI生成企划报告

### 3.3 支付状态轮询

**前端轮询机制**:
```javascript
const pollOrderStatus = (orderId: string) => {
  const interval = setInterval(async () => {
    // 查询订单状态
    const { data } = await supabase
      .from("membership_orders")
      .select("status")
      .eq("id", orderId)
      .single();
    
    if (data?.status === 'confirmed') {
      clearInterval(interval);
      // 支付成功,更新UI
    }
  }, 3000); // 每3秒查询一次
  
  // 5分钟后停止轮询
  setTimeout(() => clearInterval(interval), 5 * 60 * 1000);
};
```

**后端支付通知**:
- 微信支付成功后会回调 `/api/wechat-pay/notify`
- 后端接收回调后,更新订单状态为`confirmed`

## 六、常见问题处理

### 6.1 支付问题

#### Q1: 用户支付后,订单状态未更新?

**排查步骤**:
1. 检查微信支付回调URL是否正确配置: `https://colour-choice.art/api/wechat-pay/notify`
2. 查看Supabase日志,检查`wechat_pay_notify`表的记录
3. 手动确认订单: 进入/admin,找到对应订单,点击"确认收款"

#### Q2: 微信支付提示"参数错误"?

**可能原因**:
- `total_fee`单位错误(必须是"分")
- `notify_url`配置错误
- 微信支付商户号(MCHID)或API密钥错误

**解决方法**:
检查`.env.local`中的微信支付配置:
```
WECHAT_MCHID=你的商户号
WECHAT_APIV2_KEY=你的API密钥
WECHAT_NOTIFY_URL=https://colour-choice.art/api/wechat-pay/notify
```

### 6.2 会员权限问题

#### Q3: 用户付费后,会员权限未生效?

**排查步骤**:
1. 检查`membership_orders`表,查看订单状态是否为`confirmed`
2. 如果状态是`pending`,说明管理员未确认
3. 如果状态是`confirmed`,检查`profiles`表的`membership_type`和`membership_expires_at`字段

**手动修复**:
```sql
-- 手动开通会员
UPDATE profiles 
SET membership_type = 'basic', 
    membership_expires_at = NOW() + INTERVAL '1 year'
WHERE id = '用户UUID';
```

### 6.3 后台登录问题

#### Q4: 管理员无法登录后台?

**排查步骤**:
1. 检查`.env.local`中的`ADMIN_EMAILS`是否包含管理员邮箱
2. 清除浏览器cookie,重新登录
3. 检查服务器日志,查看`/api/admin/login` API的返回

**重置管理员密码**:
- 在Supabase Auth中,找到对应邮箱,点击"Send reset password"

### B.1 微信支付相关

#### B.1.1 统一下单API

**接口**: `/api/wechat-pay/unified-order`  
**方法**: POST  
**描述**: 调用微信支付统一下单接口,获取支付链接

**请求参数**:
```json
{
  "product_id": "商品ID",
  "product_title": "商品标题",
  "total_fee": 298000,  // 单位:分
  "quantity": 1,
  "platform": "native"  // native/mini/mp
}
```

**返回示例**:
```json
{
  "code_url": "weixin://wxpay/bizpay/xxx",  // 支付链接
  "order_no": "WX1234567890ABCDEF",
  "prepay_id": "wx20250101xxx"
}
```

**接口**: `/api/wechat-pay/query?out_trade_no=订单号`  
**方法**: GET  
**描述**: 查询微信支付订单状态

**返回示例**:

## 附录C：常用SQL查询

### C.1 查询付费用户

```sql
-- 查询所有已付费会员
SELECT 
  p.full_name,
  p.email,
  mo.plan_name,
  mo.price / 100.0 AS price_yuan,
  mo.confirmed_at
FROM profiles p
JOIN membership_orders mo ON p.id = mo.user_id
WHERE mo.status = 'confirmed'
ORDER BY mo.confirmed_at DESC;
```

### C.2 查询企划订单

```sql
-- 查询所有企划需求
SELECT 
  pr.brand_name,
  pr.season,
  pr.status,
  pp.amount / 100.0 AS paid_amount_yuan,
  pr.created_at
FROM planning_requests pr
LEFT JOIN planning_payments pp ON pr.user_id = pp.user_id
ORDER BY pr.created_at DESC;
```

### C.3 查询商品销量

```sql
-- 查询商品销量排行
SELECT 
  product_title,
  COUNT(*) AS order_count,
  SUM(quantity) AS total_quantity,
  SUM(total_amount) / 100.0 AS total_revenue_yuan
FROM buyer_orders
WHERE status IN ('paid', 'shipped', 'delivered')
GROUP BY product_id, product_title
ORDER BY total_quantity DESC
LIMIT 20;
```
